## 4.6. Domain-Driven Software Architecture.
### 4.6.1. Software Architecture Context Diagram.
1. System Context Diagram: Diagrama que muestra la relacion del aplicativo con los usuarios
se incluyen servicios externos (si hay pocos bounded context se incluyen ahi)

2. Bounded Context Map: Muestra la relacion entre bounded contexts (los bounded context son como una burbuja que encapsula palabras clave en los procesos para poder diferenciarlos [Ej. Bounded context enfocado en las ventas(ventas), otro en revisar el stock(gestion) y otro enfocado en los proveedores (suministros)]) Se hace como un brainstorm y se ve en que pueden conectarse o comunicarse [se usa un circulo entre conexiones lineales (upstream o downstream) para definir comunicacion, algunos context se pueden integrar para representar por ejemplo un share model por database, tambien se mencionan los (third party context para definir los restful apis)]
   
### 4.6.2. Software Architecture Container Diagrams.
1. Bounded Context Deployable / Container Diagrams. Sirve para entender como funciona y el proceso, Se especifican DB's, indexers, Search engine, las Apis que usen los bounded context y se conectan por flechas, las cuales tienen como objetivo explicar la direcion y relacion junto a que se esta enviando/comunicando (TCP) 

### 4.6.3. Software Architecture Components Diagrams.
1. Component diagrams: Estos van a mostrar las ordenes, procesos, mensajes y componentes utilizados en el uso del aplicativo, claro se deben hacer diferentes de estos para cada bounded o USER GOALS

#### 5.2.1.6. Services Documentation Evidence for Sprint Review.
Durante el Sprint 1 de IoBuild, el desarrollo se centró exclusivamente en la implementación de la interfaz gráfica, con énfasis en el diseño, maquetación e internacionalización de la landing page. En esta primera etapa, no se trabajó aún en servicios backend ni en endpoints API REST, por lo que no corresponde incluir documentación técnica de servicios.

**Estado actual:**

- No se han implementado controladores ni servicios de aplicación.

- No se han definido endpoints HTTP ni documentación en Swagger u OpenAPI.

- No se han activado microservicios ni lógica de negocio en la capa backend.

- El proyecto se encuentra en etapa de diseño visual, accesibilidad (a11y), i18n y diseño responsivo.

Justificación:
El enfoque de este sprint responde a las épicas relacionadas con la Landing Page, Internacionalización, Responsive UI y Accesibilidad, las cuales son fundamentales para presentar la propuesta de valor de IoBuild, pero no requieren aún integración con lógica de negocio.

#### 5.2.1.8. Team Collaboration Insights during Sprint.
Durante el Sprint 1 de IoBuild, las actividades se organizaron en torno a la Landing Page, enfocando los esfuerzos en el diseño visual, la accesibilidad y la internacionalización inicial. Cada miembro del equipo asumió tareas específicas, asegurando una participación equilibrada en la construcción de la base del producto.

En primer lugar, se trabajó en la estructura visual y el diseño responsivo, utilizando CSS para garantizar que la plataforma se adaptara correctamente a distintos dispositivos. Se implementaron secciones clave como “Sobre Nosotros”, “Preguntas Frecuentes (FAQ)”, "Beneficios", "Planes", "Caracteristicas"

Posteriormente, se añadió contenido esencial a las secciones desarrolladas, definiendo la forma en que la información sería presentada y asegurando coherencia con la propuesta de valor de IoBuild. Para mejorar la experiencia visual, se integraron imágenes representativas, incluyendo un hero image, gráficos de servicios principales y otras imágenes de apoyo.

Además, se incorporaron los primeros avances de internacionalización (i18n) mediante lógica en JavaScript, lo que permitió habilitar la traducción al español en todas las secciones

La colaboración del equipo se gestionó a través de GitHub, aplicando un flujo de trabajo basado en ramas por feature, commits con la convención Conventional Commits y Pull Requests revisados en equipo, lo que aseguró trazabilidad y calidad en las entregas.
